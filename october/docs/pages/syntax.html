arrow_backScript Syntax

Scripts 101

You can describe different types of scenes for your games, they all follow similar syntax but all have exceptions.

Types of scenes:


Description:


Show the player a text, if the user clicks on the screen all the text appears at once, otherwise it appears slowly.
Once all the text is visible the user can click to end the scene.

Syntax:

{

	"type":<"text"> (Scene type),

	"background":<Filename or "none"> (optional, Background image),

	"musicStart":<Filename> (optional, Start or change the background music),

	"musicStop":<Empty String> (optional, Stop the background music),

	"soundPlay":<Filename> (optional, Play a sound),

	"color":<rgba() or color name> (optional, Text color),

	"text":<String> (Text to show to the player),

	"localization"<Localization object Array> (Optional, localized text),

	"key":<Variable Name> (optional, userStorage key),

	"value":<String> (optional, Assign a value to "key"),

	"add":<String> (optional, Add a value to "key"),

	"substract":<String> (optional, Substract a value to "key"),

	"target":<Script Identifier or Variable Name>(optional,Script to be executed when this scene ends),

	"branching":<Branching Object>(optional, Allows different targets and outcomes based on variable values),

	"image":{

		"name":<Filename>(Image Filename),

		"width":<Number>(Image Width),

		"height":<Number>(Image Height)

	},

	"checkPoint":<Boolean>(optional, if true the current state of userStorage gets stored on disk),

	"noFadeIn":<Boolean>(optional, if true the fadeIn effect gets disabled),

	"noFadeOut":<Boolean>(optional, if true the fadeOut effect gets disabled)

}

Description:


Present the player a choice, buttons can be shown or not using conditions
Once all the player selects a choice the scene ends.

Syntax:

{

	"type":<"choice"> (Scene type),

	"background":<Filename or "none"> (optional, Background image),

	"musicStart":<Filename> (optional, Start or change the background music),

	"musicStop":<Empty String> (optional, Stop the background music),

	"soundPlay":<Filename> (optional, Play a sound),

	"choices":<Choice Object Array> (Choices definition),

	"checkPoint":<Boolean>(optional, if true the current state of userStorage gets stored on disk),

	"noFadeIn":<Boolean>(optional, if true the fadeIn effect gets disabled)

}

Choice Object Syntax:

{

	"type":<"redirect" or "exit"> (Choice type, exit is only available on desktop),

	"condition":<"Condition Object"> (Optional,Condition required to show this choice),

	"text":<String> (Text to show to the player),

	"localization"<Localization object Array> (Optional, localized text),

	"language"<String> (Optional, changes the current language of the game),

	"key":<Variable Name> (optional, userStorage key),

	"value":<String> (optional, Assign a value to "key"),

	"add":<String> (optional, Add a value to "key"),

	"substract":<String> (optional, Substract a value to "key"),

	"target":<Script Identifier or Variable Name>(optional,Script to be executed when this scene ends),

	"branching":<Branching Object>(optional, Allows different targets and outcomes based on variable values),

	"noFadeOut":<Boolean>(optional, if true the fadeOut effect gets disabled)

}

Description:

If the time passes or if the scene is skippeable and the user click the screen the scene ends.

Syntax:

{

	"type":<"wait"> (Scene type),

	"background":<Filename or "none"> (optional, Background image),

	"musicStart":<Filename> (optional, Start or change the background music),

	"musicStop":<Empty String> (optional, Stop the background music),

	"soundPlay":<Filename> (optional, Play a sound),

	"timeout":<Number> (Number of milliseconds before the scene ends),

	"skippeable":<Boolean> (If set to true the scene can be skipped),

	"key":<Variable Name> (optional, userStorage key),

	"value":<String> (optional, Assign a value to "key"),

	"add":<String> (optional, Add a value to "key"),

	"substract":<String> (optional, Substract a value to "key"),

	"target":<Script Identifier or Variable Name>(optional,Script to be executed when this scene ends),

	"checkPoint":<Boolean>(optional, if true the current state of userStorage gets stored on disk),

	"noFadeIn":<Boolean>(optional, if true the fadeIn effect gets disabled),

	"noFadeOut":<Boolean>(optional, if true the fadeOut effect gets disabled)

}

Description:


It can be use to create click and point scenes.
The scene ends when the player clicks an sprite with target or branching.

Syntax:

{

	"type":<"interactive"> (Scene type),

	"background":<Filename or "none"> (optional, Background image),

	"musicStart":<Filename> (optional, Start or change the background music),

	"musicStop":<Empty String> (optional, Stop the background music),

	"soundPlay":<Filename> (optional, Play a sound),

	"checkPoint":<Boolean>(optional, if true the current state of userStorage gets stored on disk),

	"noFadeIn":<Boolean>(optional, if true the fadeIn effect gets disabled),

	"noFadeOut":<Boolean>(optional, if true the fadeOut effect gets disabled),

	"sprites":<Sprite Object Array>(Sprite Array)

}

Sprite Object Syntax:

{

	"img":<Filename>(Sprite Filename),

	"position":{

		"x":<Number>(Sprite position from the left),

		"y":<Number>(Sprite position from the top)

	},

	"size":{

		"x":<Number>(Sprite Width),

		"y":<Number>(Sprite Height)

	},

	"zindex":<Number>(Sprite Depth),

	"cursor":<cursor>(Cursor type that shows when the player hover over the sprite),

	"key":<Variable Name> (optional, userStorage key),

	"value":<String> (optional, Assign a value to "key"),

	"add":<String> (optional, Add a value to "key"),

	"substract":<String> (optional, Substract a value to "key"),

	"noFadeOut":<Boolean>(optional, if true the fadeOut effect gets disabled),

	"branching":<Branching Object>(optional, Allows different targets and outcomes based on variable values),

	"target":<Script Identifier or Variable Name>(optional,Script to be executed when this scene ends)

}

Features:

Description:


Condition that gets evaluated by the engine and used to determined if a choice should be shown or a branch should be followed.

Syntax:

{

	"key":<Variable Identifier> (userStorage key),

	"operator":<"lessThan","moreThan","isSet","isNotSet","notEqual" o "equal">,

	"value":<String> (value to operate, optional when using "isSet" or "isNotSet" as operator)

}

Description:


Branching evaluates a condition object, if its true then the first branch gets interpreted else the second branch gets executed.

Syntax:

{

	"condition":<Condition Object>(Condition to evaluate),

	"target":<Script or Variable Identifier>(optional, Script to execute if the condition is true),

	"targetBranching":<Branching Object> (optional, similar to an and if statement),

	"default":<Script or Variable Identifier>(optional, Script to execute if the condition is false, similar to an else statement),

	"defaultBranching":<Branching Object> (optional, similar to an else if statement)

}

Description:


Allows to localizate text of the game so it can be played in different languages

Syntax:

{

	"language":<String>(Language of the text),

	"text":<Script>(localized text),

}
